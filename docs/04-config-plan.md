## 2. 分支策略

| 分支 | 用途 | 说明 |
|---|---|---|
| `main` | 最终提交分支 | 仅接受来自 `dev` 的合并，存放可交付的稳定版本 |
| `dev` | 开发集成分支 | 各 `feature-*` 分支合并的目标分支，用于阶段性集成 |
| `feature-*` | 个人功能分支 | 每位成员独立工作的分支，完成后合并至 `dev` |

分支生命周期：

```text
main ◄── dev ◄── feature-memberA
            ▲── feature-memberB
            ▲── feature-memberC
            ▲── feature-memberD
```

- 成员在各自 `feature-*` 分支上完成分配任务
- 完成后提交 Pull Request / 合并请求至 `dev`
- 配置管理员审核后合并至 `dev`
- 全部工作完成并验证后，由配置管理员将 `dev` 合并至 `main`

## 3. 命名约定

### 3.1 分支命名

| 类型 | 格式 | 示例 |
|---|---|---|
| 个人分支 | `feature-<功能名称>` | `feature-wbs`|
| 临时修复 | `fix-<简短描述>` | `fix-readme-typo` |

### 3.2 文档命名

| 规则 | 示例 |
|---|---|
| 文档使用 `##-英文短名.md` 格式，数字两位 | `01-project-charter.md` |
| 一律使用小写字母与连字符 `-`，不使用空格或下划线 | `02-wbs.md` |

### 3.3 提交信息约定

提交信息采用以下格式：

```text
<类型>: <简短描述>

类型包括：
  无      无具体功能或文档修改
  feat    新功能 / 新文档
  fix     修复 / 更正
  docs    纯文档修改
  merge   合并操作

示例：
  feat: 完成 WBS 三级分解
  fix: 修正 README 成员分工表
  docs: 补充配置管理文档命名约定
  merge: 将 feature-zhangsan 合并至 dev
```

### 3.4 目录结构约定

│   ├── 01-project-charter.md

## 4. 合并规则

### 4.1 合并流程

1. 成员在 `feature-*` 分支完成工作并自检通过
2. 提交前确保已拉取 `dev` 最新提交并在本地解决冲突
3. 提交 Pull Request 或直接请求配置管理员合并
4. 配置管理员（成员C）负责审查并执行合并至 `dev`
5. 全部交付物齐备后，配置管理员将 `dev` 合并至 `main`
6. `main` 分支禁止直接推送，仅通过合并进入

### 4.2 冲突处理

1. 发生冲突时，冲突双方共同在本地对比差异
2. 依据项目目标协商确定最终内容
3. 手动编辑冲突标记，完成合并
4. 在 `docs/conflict.md` 中简要记录冲突内容与解决方式
5. 提交合并结果，提交信息使用 `merge:` 前缀

## 5. 版本留痕

### 5.1 关键提交标记

每个阶段性成果完成后，使用 Git Tag 标记版本：

| 标签 | 里程碑 | 预计提交分支 |
|---|---|---|
| `v0.1-init` | 项目初始化完成（README + 目录结构） | `dev` |
| `v0.2-wbs` | WBS 三级分解完成 | `dev` |
| `v0.3-schedule` | 进度计划形成 | `dev` |
| `v0.4-config` | 配置管理文档完成 | `dev` |
| `v0.5-summary` | 总结报告完成 | `dev` |
| `v1.0-final` | 全部交付物合并至 main | `main` |

打标签命令示例：
```bash
git tag -a v0.1-init -m "项目初始化完成"
git push origin v0.1-init
```

### 5.2 提交历史保留

- 所有分支合并均使用 `git merge --no-ff`，保留分支历史轨迹
- 关键节点使用 Git Tag 留痕，便于回溯
- `git log --graph --oneline --all` 可查看完整版本演进图

### 5.3 冲突记录

所有合并冲突及其解决方式统一记录在 `docs/conflict.md`，内容包括：

- 冲突发生的文件
- 冲突原因简述
- 协商后的解决方案
- 参与解决的人员
