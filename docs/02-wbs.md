## 3. 分解说明

### 分解思路

本 WBS 采用 **以交付物为导向 + 按生命周期阶段** 相结合的分解方式，具体思路如下：

1. **项目管理成果（1）** — 按照项目生命周期（启动→计划→配置→总结）纵向分解，涵盖课程实验要求的全过程文档产出，确保每个阶段有明确的交付物。

2. **业务功能成果（2）** — 围绕系统的三大核心业务（活动、报名、签到）横向分解，每个业务模块独立为二级节点，便于团队成员并行开发和分工协作（马永超负责计划相关、林立天负责配置管理相关）。

3. **技术实现成果（3）** — 按技术栈分层分解（数据库→前端→后端→测试），确保每个技术层次有具体的交付成果，便于进度跟踪和质量检查。

4. **部署与提交成果（4）** — 作为收尾阶段，涵盖环境配置和版本控制，确保最终成果可运行、可提交。

### 划分依据

- **一级按阶段和领域划分**，使各交付物归属清晰，避免交叉重叠。
- **三级节点为最小可交付单元**，每个节点均可独立验证和分配责任人。
- **业务模块独立分解**，有利于识别各模块间的依赖关系（如：活动管理是报名和签到的基础）。
