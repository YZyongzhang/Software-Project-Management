# 01 项目启动说明

## 1. 项目名称
校园活动报名与签到管理系统

## 2. 项目目标
- 支持活动发布
- 支持学生报名
- 支持签到与统计

## 3. 团队分工
| 姓名 | 角色 | 主要职责 |
|---|---|---|
| 葛团辉 | 项目经理 | 统筹进度、最终提交和冲突控制解决 |
| 马永超 | 计划负责人 | 负责 WBS 与进度计划文档 |
| 林立天 | 协作成员 | 参与文档完善，分支提交与冲突解决 |

## 4. 项目边界
### 纳入范围
1. 项目仓库与目录结构
2. WBS 文档
3. 简化版进度计划文档
4. 配置管理文档
5. 总结报告
6. 活动信息发布与维护
7. 学生报名信息提交
8. 报名名单查看与审核
9. 签到记录与签到状态管理
10. 签到统计结果导出
### 不纳入范围
1. 复杂支付功能
2. 与真实校园统一身份认证系统的深度对接
3. 高并发优化与正式部署方案
4. 移动端 App 正式开发

## 5. 主要交付物
- 项目仓库
- WBS 文档
- 进度计划文档
- 配置管理文档
- 总结报告
